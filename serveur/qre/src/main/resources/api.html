/authentication

HTTP Methods Path Details
POST If login and password match a 'professeur', generate a new token

/classes

HTTP Methods Path Details
GET Get all classes
GET /{classe_id} Get classe by id
GET /{classe_id}/groupes Get all groupes in classe by id
POST Insert new classe
PUT /{classe_id} Update classe by id
DELETE /{classe_id} Delete class by id

/groupes

/qrcode

HTTP Methods Path Details
GET /scan/{token_generated} Try to read a cookie to identify the 'etudiant' and his 'signature' updates for 'emergement' with 'token_generated'
POST /authentication_etudiant If login and password match an 'etudiant', set a cookie with num_etu
